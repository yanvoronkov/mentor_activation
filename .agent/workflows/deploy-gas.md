# Workflow: Деплой GAS проекта

## Вариант 1: Использование CLASP (Рекомендуется)
Если настроен инструмент `clasp`:

1. Убедитесь, что все изменения сохранены локально.
2. Выполните команду пуша кода:
   `clasp push`
3. Откройте проект в браузере для деплоя новой версии:
   `clasp open`
4. В редакторе GAS:
   - Нажмите "Deploy" -> "New deployment".
   - Выберите тип "Web app".
   - Установите новую версию (Description: описание изменений).
   - Нажмите "Deploy".
   - Скопируйте новый URL (если он изменился, обычно для "Web App" URL остается тем же для `exec`, но версия обновляется).

## Вариант 2: Ручное копирование
1. Скопируйте содержимое `.gs` файлов из локального проекта.
2. Вставьте его в соответствующие файлы в редакторе Google Apps Script.
3. Сохраните проект (Cmd/Ctrl + S).
4. Выполните шаги по деплою (Deploy -> New deployment) как описано выше.

## Важное примечание
При изменении `doGet`/`doPost` или добавлении новых scope всегда требуется **New Deployment** для обновления прав доступа и версии скрипта. Простого сохранения ("Save") недостаточно для обновления Web App URL endpoint.
