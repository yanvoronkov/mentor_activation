# Базовые правила проекта (Vanilla JS + GAS)

Этот файл определяет основные архитектурные принципы и технологический стек проекта. Он наследует глобальные стандарты пользователя.

## Технологический стек
- **Frontend**: Native HTML5, CSS3, ES6+ JavaScript.
- **Backend**: Google Apps Script (GAS).
- **Database**: Google Sheets.
- **Communication**: REST API (JSON via HTTPS).

## Основные принципы
- **No Heavy Libraries**: Запрещено использование тяжелых библиотек и фреймворков (jQuery, React, Bootstrap, Tailwind). Используется только нативный код для максимальной производительности.
- **Производительность**: Минимизация количества запросов, оптимизация ассетов и быстрая загрузка — приоритет №1.
- **Разделение ответственности**: 
  - Frontend отвечает за отображение, UI/UX и валидацию на стороне клиента.
  - Backend (GAS) отвечает за бизнес-логику, безопасность и работу с данными в Google Sheets.
  - Обмен данными происходит строго через JSON.

## Структура проекта
- `/index.html` — главная точка входа фронтенда.
- `/styles.css` — основные стили.
- `/google-apps-script-example.js` — пример бэкенд-кода (будет разделен на модули).
- `/backend/` — исходный код GAS.

## Кодирование и стандарты
- Весь текст, комментарии и документация — на **русском языке**.
- Имена переменных и функций — на английском.
- Использование `strict mode` во всех JS файлах.
- Использование современных стандартов ES6+ (arrow functions, template literals, destructuring).
