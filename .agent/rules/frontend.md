# Правила разработки Frontend (Vanilla JS)

## HTML и Верстка
- **Семантика**: Использование семантических тегов HTML5 (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`).
- **Структура**: Чистая и понятная иерархия элементов. Каждая страница должна иметь один заголовок `<h1>`.
- **Доступность**: Использование атрибутов `aria-` и `alt` для изображений.

## CSS и Стилизация
- **Layout**: Использование Flexbox и CSS Grid для построения сеток и выравнивания.
- **Стилистика Modern Dashboard**:
  - Glassmorphism: использование `backdrop-filter: blur()` и полупрозрачных фонов.
  - Мягкие тени (`box-shadow`) и скругленные углы (`border-radius`).
- **Анимации**: Использование плавных CSS-переходов (`transition`) и ключевых кадров (`@keyframes`) для микровзаимодействий (`fadeIn`, `slideDown`).
- **Адаптивность**: Mobile-first подход с использованием `@media` запросов.

## JavaScript (ES6+)
- **Запросы**: Использование Native Fetch API для взаимодействия с GAS API.
- **Синтаксис**:
  - Использование `const` и `let` вместо `var`.
  - Стрелочные функции (`() => {}`).
  - Деструктуризация объектов и массивов.
  - Шаблонные строки (` `text ${var}` `).
  - `async/await` для обработки асинхронных операций.
- **Параметры URL**: Использование `URLSearchParams` для чтения и записи параметров в адресной строке.
- **Безопасность**:
  - КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО использовать `innerHTML` для вставки данных от пользователя. Используйте `textContent` или `innerText`.
  - Валидация всех входящих данных перед отправкой на бэкенд.
